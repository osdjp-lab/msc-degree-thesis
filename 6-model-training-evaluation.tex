\section{Model training and evaluation}\label{Model training and evaluation}

This section presents an overview of the primary guiding principals used during the stages of model training and evaluation.

\subsection{Bias-Variance dilemma}\label{Bias-Variance dilemma}

The bias-variance dilemma is a fundamental problem in machine learning and involves the issues of model optimization and generalization. In the case of optimization (reducing the model error) a model eventually starts to overfit the training data, this results in high error of the models predictions on the test/validation datasets (high variance). The opposite is aiming to increase model generalization (reduction of model complexity) however this in turn causes underfitting or high error of the models predictions on the training dataset (high bias). This counterpoised behaviour of both bias and variance enables the easy assessment of model suitability for use in the prediction of unseen data \cite{ibm-ensemble,ibm-regularization,svm-2}.

\subsection{Performance metrics}\label{Performance metrics}

The accuracy of the model forecasts was evaluated on the basis of commonly used measures including:
\begin{itemize}
    \item \Glsxtrlong{mse} - measures the average squared difference between predicted and actual values, providing a sense of the magnitude of the errors. A lower MSE indicates better fit, but it is sensitive to outliers due to the squaring of errors \cite{scikit-mse}.

        $$MSE = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$

        Where:
        \begin{itemize}
            \item $n$ is the total number of predictions
            \item $y_i$ is the actual value at time $i$
            \item $\hat{y}_i$ is the predicted value at time $i$
        \end{itemize}

    \item \Gls{ds} - also known as the hitrate is a metric that measures the ability of a forecasting model to correctly predict the direction of change in the target variable. It calculates the percentage of times the model correctly predicts whether the value will increase or decrease compared to the previous period \cite{svm-2}.

        $$C = \sum_{i=1}^{n} \begin{cases}
        1 & \text{if } (y_i - y_{i-1}) \text{ and } (\hat{y}_i - \hat{y}_{i-1}) \text{ have the same sign} \\
        0 & \text{otherwise}
        \end{cases}$$

        $$DS = \frac{C}{n} \times 100$$

        Where:
        \begin{itemize}
            \item $C$ is the number of correct directional predictions
            \item $n$ is the total number of predictions
            \item $y_i$ is the actual value at time $i$
            \item $\hat{y}_i$ is the predicted value at time $i$
        \end{itemize}

The DS metric measures the percentage of times the model correctly predicts the direction of change in the target variable.

    \item \Gls{oob} error - is a method for evaluating the performance of ensemble methods where during training on a random subset of the training data the generated model is evaluated on the data that was not part of its training dataset. It provides an unbiased estimate of the model's generalization performance without the need for a separate validation set \cite{scikit-oob}.
\end{itemize}


\subsection{Methodology}\label{Methodology}

The process of model training and evaluation consisted of the selection of the optimal data processing methodology and offset followed by hyperparameter optimization. In both cases each models accuracy was evaluated on both in-sample (training set) and out-of-sample (test set) data. The minimum average \gls{mse} and maximum average \gls{ds} values of the performance metrics for the training and test set  were selected as exhibiting the best performance.

\subsubsection{Processing methodology and offset selection}\label{Processing methodology and offset selection}

In the first step the performance of each of the models on each of the dataset processing variations was evaluated. Each model was used with its default parameters with some exceptions. For the \gls{mlp} the  activation function was changed to \gls{tanh} due to the normalization range of the data, shuffle was set to false to prevent data shuffling in order to not disrupt the continuous time series nature of the data, random state was set to zero for reproducability of the results and the maximum number of iterations was increased to 1000 in order to allow for model convergence. For the \gls{rf} only the random state was set. For the \gls{svr} non of the default parameters were changed.

In the case of the \gls{mlp} for the :
\begin{itemize}
    \item \gls{mse} as can be seen in figure \ref{fig:mlp-mse} the best result was found for the differenced and normalized dataset at an offset of 11 days.
    \item \gls{ds} as can be seen in figure \ref{fig:mlp-ds} the best result was found for the differenced and normalized dataset at an offset of 40 days.
\end{itemize}

In the case of \gls{svr} for the :
\begin{itemize}
    \item \gls{mse} as can be seen in figure \ref{fig:svr-mse} the best result was found for the log transformed and differenced dataset at an offset of 9 days.
    \item \gls{ds} as can be seen in figure \ref{fig:svr-ds} the best result was found for the differenced and normalized dataset at an offset of 28 days.
\end{itemize}

In the case of \gls{rf} for the :
\begin{itemize}
    \item \gls{mse} as can be seen in figure \ref{fig:rf-mse} the best result was found for the log transformed and differenced dataset at an offset of 21 days.
    \item \gls{ds} as can be seen in figure \ref{fig:rf-ds} the best result was found for the differenced and normalized dataset at an offset of 49 days.
\end{itemize}

\subsubsection{Hyperparameter optimization}\label{Hyperparameter optimization}

With the processing methodology and offset size selected in the previous step for each model a grid search with 5-fold cross-validation utilizing the negative \gls{mse} loss function was conducted for a selected set of parameters for each model. First to better understand the dynamics of each model the values of different parameters were manipulated independantly. In the second step a broader grid search was conducted on the same set of ranges for each of the variables but for every possible combination of values.

For the \gls{mlp} regression model four parameters were varied: hidden layer sizes, alpha, solver and tol. In the case of the independant analysis the size of the hidden layer had a small effect on the accuracy of the model with the greatest error being within the ranges 0 - 10 and 50 - 90 and the smallest for 93 as can be seen in figure \ref{fig:mlp-hidden}. The accuracy of the model was greater for values of alpha above 0.2 however larger values provided a higher accuracy as such the optimal value of alpha was 0.99 as is shown in figure \ref{fig:mlp-alpha}. For the solver the \gls{lbfgs} provided the best accuracy when compared against adam and \gls{sgds} as is shown in figure \ref{fig:mlp-solver}. For the tolerance above the value of 0.00023 the accuracy reaches it's maximum and remains nearly constant upto the end of the search range at 0.0007 as can be seen in figure \ref{fig:mlp-tol}. In the case of the full search space grid search the results were different and are presented in table \ref{table:mlp}.
