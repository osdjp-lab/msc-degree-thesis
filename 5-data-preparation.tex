\section{Data preparation}\label{Data preparation}

In this paper data preparation for use in \gls{ai} model creation consists of two steps: acquisition and preprocessing.

\subsection{Acquisition}\label{Acquisition}

The data chosen for this study was a set of euro foreign exchange reference rates obtained from the website of the \gls{ecb} \cite{forex-data}. The data consists of the daily spot prices for 41 foreign currencies from 04.01.1999 to 02.05.2025. The currencies included in the dataset were: \gls{usd}, \gls{jpy}, \gls{bgn}, \gls{cyp}, \gls{czk}, \gls{dkk}, \gls{eek}, \gls{gbp}, \gls{huf}, \gls{ltl}, \gls{lvl}, \gls{mtl}, \gls{pln}, \gls{rol}, \gls{ron}, \gls{sek}, \gls{sit}, \gls{skk}, \gls{chf}, \gls{isk}, \gls{nok}, \gls{hrk}, \gls{rub}, \gls{trl}, \gls{try}, \gls{aud}, \gls{brl}, \gls{cad}, \gls{cny}, \gls{hkd}, \gls{idr}, \gls{ils}, \gls{inr}, \gls{krw}, \gls{mxn}, \gls{myr}, \gls{nzd}, \gls{php}, \gls{sgd}, \gls{thb}, \gls{zar}.

\subsection{Preprocessing}\label{Preprocessing}

The cleaning and preparation of the data was conducted in order to increase the accuracy of predictions as well as improve the efficiency of model training.

\subsubsection{Handling of missing values}\label{Handling of missing values}

The handling of missing values consisted of two steps.

The first step in the preparation of the dataset was the removal of variables with more than two sequential variable specific missing values, caused by insufficient market activity and reporting inconsistencies, of which there were 24, leaving behind 17 currency exchange rates: \gls{usd}, \gls{jpy}, \gls{czk}, \gls{dkk}, \gls{gbp}, \gls{huf}, \gls{pln}, \gls{sek}, \gls{chf}, \gls{nok}, \gls{aud}, \gls{cad}, \gls{hkd}, \gls{krw}, \gls{nzd}, \gls{sgd}, \gls{zar}.

Even though variables with variable specific missing values were eliminated in the previous step and the remaining data appeared continuous the remaining exchange rates still exhibited global missing values which were the result of holidays and weekends, days where the exchange markets were closed and no value could be assigned. In order to alleviate this issue and preserve the continuity of the time series the missing dates were added and the missing values were forward filled with the values preceding them.

This step was crucial, as missing data can lead to biased estimates and can negatively affect the overall performance of predictive models \cite{techniques,rf-svr}.

\subsubsection{Feature scaling}\label{Feature scaling}

Feature scaling is an essential step in the data preparation for many \gls{ai} models which often exhibit better training efficiency and forecasting accuracy with scaled data. This is especially true for \gls{mlp} and \gls{svr}. For the \gls{rf} machine learning model this step is not required due to insensitivity and as a result has been omitted. Two feature scaling methods where used in the data preparation process for different models in this paper. An visual representation of the effects of both can be seen in figure \ref{fig:usd-scaling}.

\begin{itemize}
    \item Normalization via scikit-learn's MinMaxScaler \cite{scikit-learn} to the range [-1,1] was used in the preparation of data for use with the \gls{mlp} with the \gls{tanh} activation function \cite{nn-vs-chaotic,ann-2}.
    \item Standardization via scikit-learn's StandardScaler \cite{scikit-learn} to a mean of zero and a variance of one was used in the data preparation pipeline for use with \gls{svr} \cite{genetic,rf-svr}.
\end{itemize}

\subsubsection{Feature extraction}\label{Feature extraction}

From among all of the available currencies the \gls{usd} currency exchange rate was selected as the target variable. For each feature variable as well as the target variable time lagged versions upto 500 days were created for each dataset preprocessing variation \cite{rf-svr,gold}.

\subsubsection{Feature decorrelation}\label{Feature decorrelation}

In the next step a Pearson correlation matrix was constructed on the feature and target exchange rates to assess the linear relationships among them. The threshold for feature rejection was set for values $\ge 0.8$. This threshold was chosen to mitigate the risk of multicollinearity, which can lead to inflated standard errors and unreliable coefficient estimates in regression models. The analysis began with the elimination of feature variables that exhibited a high linear correlation with the target variable. Next feature variables that had a high correlation with at least one other variable were removed. This was done in order to prevent amplifying or distorting effects during model training \cite{genetic}. As a result of this analysis, the initial feature set consisting of 8500 variables was significantly reduced down to 39 variables.

\subsubsection{Variable selection}\label{Variable selection}

\subsubsection{Data split}\label{Data split}

For the purpose of model training and forecasting accuracy evaluation each dataset was split into two subsets with a separation strip equal to the degree of generated time lags (150) in order to prevent information leakage from the training set to the test set:
\begin{itemize}
    \item training $+$ validation (70\%) - for model training and k-fold cross-validation.
    \item testing (30\%) - for the final evaluation of the selected model in order to prevent overfitting and enable a unbiased evaluation of the model on new unseen data.
\end{itemize}
